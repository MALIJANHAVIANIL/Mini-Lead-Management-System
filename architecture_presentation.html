System Design

1. Project Architecture

02 / 08

The application is built on a **fully decoupled Client-Server architecture**, isolating backend resources from client view layers:

  • Thin-Client Frontend: Built using semantic HTML5, clean vanilla CSS variables, and a client-side Hash-based SPA routing controller.
  • Express.js API: A RESTful backend that handles core routing, inputs validation, JWT checks, and database interactions.
  • WebSocket Broadcast Server: Lives on the HTTP channel and pushes instant event alerts (e.g., creations, modifications) to active client dashboards.

Security

3. Authentication & RBAC

04 / 08

We implement secure, stateless JWT Authentication combined with detailed Role-Based Access Control (RBAC):

  • Token Guards: Incoming client API requests verify JWT signatures in a middleware wrapper, translating sub-identifiers into user profiles.
  • Admin / Manager Rights: Full permission sets. Access to overall creation, modifications, custom assignments, and logs.
  • Agent Scope Restrictions: Agents can only fetch leads assigned to them. At update time, attempts to modify structural attributes (e.g. client names, assignments) throw a 403 Forbidden response. Agents can only alter status and notes.

4. Database Design & Duality

05 / 08

Our persistence tier is designed for absolute flexibility and optimized for performance metrics:

  • Dual-Engine Compatibility: By default, the app initializes zero-config local SQLite. Simply adding a connection string in DATABASE_URL dynamically binds the application to PostgreSQL.
  • Normalized Relations: Users reference leads through foreign keys. Leads write history rows to a separate activity logs auditor table (cascade-deleted with leads).
  • Index Optimization: High-cardinality filters and assignments utilize database indexes (status, source, assigned_to) to maintain sub-millisecond retrieval speeds.

5. Auto-Assignment Logic

06 / 08

We implemented a workload-balanced **Least-Loaded Agent** selector algorithm, executed in a single DB query:

  • Workload Aggregation: The scheduler counts total open leads held by active agents.
  • SQL-Driven Decision: Left joins and group aggregations fetch the target agent in a single database roundtrip, optimizing resource usage.
  • Graceful Degradation: If no agents are registered in the system, the lead remains unassigned rather than failing, allowing manual overrides later.
-- Workload Distribution Query
SELECT u.id, u.name, COUNT(l.id) AS lead_count
FROM users u
LEFT JOIN leads l ON u.id = l.assigned_to
WHERE u.role = 'Agent'
GROUP BY u.id
ORDER BY lead_count ASC
LIMIT 1
Performance

6. Scalability & Extensibility

07 / 08

Our code includes non-functional parameters to safeguard performance at high usage scales:

  • Offset Pagination: All listings support server-enforced pagination (using database cursors via SQL LIMIT & OFFSET), preventing memory bloat on heavy tables.
  • Client-Side Debouncing: Live-searching inputs use a 400ms debounce buffer, filtering duplicate HTTP fetch calls during active user keypresses.
  • Third-Party Enrichment Scraper: Fetches domain enrichments asynchronously using microlink.io. Implements a 4-second maximum fetch timeout to ensure database lead creation succeeds even if the external scraper lags.
🚀 Pagination Mode: Backend-enforced LIMIT/OFFSET database cursors.
⏱️ Scraper Isolator: Non-blocking fetch calls with strict fallback timeouts.
📉 Debounced Fetches: Prevents rapid frontend typing API floods.

7. Challenges & Future Scope

08 / 08

During the build, we encountered platform and design hurdles, mapping clear mitigations and extensions:

  • PowerShell Restrictions: PowerShell execution controls blocked default npm scripts. We bypassed this by invoking npm.cmd directly inside the command runtime environment.
  • Future caching: To optimize heavy workloads, we plan to implement a Redis store layer to cache dashboard summaries and active list metrics.
  • Real-time Notifications: The WebSocket setup can be combined with Nodemailer worker services to email agents immediately upon assignment.
